PHASE 1 — BUG FIXES & SECURITY HARDENING
================================================================================

1) CRITICAL BUG: Editing a note corrupted the notes list (app crash)
   File: notesphere/src/components/notes/NoteView.js
   - Removed the setNotes({ ...notes, title, description, tag }) call inside
     handleClick(). It spread an ARRAY into an OBJECT, turning `notes` into a
     non-array and crashing notes.filter(...) on every edit (white screen).
   - Removed the now-unused `setNotes, notes` from the context destructure.
   - editNote() already updates the array correctly, so editing still works.

2) CRITICAL BUG: Adding a note set state to `undefined`
   File: notesphere/src/context/notes/noteState.js
   - addNote() did setNotes(json.notes) but the backend returns { savedNote }.
   - Changed to: setNotes((prevNotes) => [...prevNotes, json.savedNote])
     so the new note is appended safely and state is never undefined.

3) BUG: Server-side length validators did nothing
   Files: notesphere-backend/routes/auth.js, notesphere-backend/routes/notes.js
   - express-validator's isLength() needs an options object, but code passed a
     bare number, e.g. isLength(3) -> ignored (min length 0), so short input
     was never rejected.
   - Fixed to isLength({ min: 3 }) / isLength({ min: 5 }) for name, password,
     note title and description.

4) SECURITY: JWTs never expired
   File: notesphere-backend/routes/auth.js
   - Added { expiresIn: "7d" } to jwt.sign(...) in BOTH /createuser and /login.

5) SECURITY: CORS wide open + no PORT fallback
   File: notesphere-backend/index.js
   - app.use(cors()) -> app.use(cors({ origin: process.env.CLIENT_URL ||
     "http://localhost:3000" }))
   - Added dotenv.config() here so CLIENT_URL/PORT are read.
   - const port = process.env.PORT || 5000 (was logging "localhost:undefined").

6) MAINTAINABILITY: Frontend API URL hardcoded in 3 places
   New file: notesphere/src/config.js
     const API_URL = process.env.REACT_APP_API_URL || "http://localhost:5000";
   - Wired API_URL into:
       notesphere/src/context/notes/noteState.js   (host)
       notesphere/src/components/authentication/Login.js
       notesphere/src/components/authentication/SignUp.js
   - Switching environments is now a single env var instead of editing 3 files.

7) CODE QUALITY: Fragile setOneNote pattern
   Files: notesphere/src/components/notes/NoteView.js
          notesphere/src/components/notes/Notesitems.js
   - setOneNote({ oneNote: null }) -> setOneNote(null) (the old form worked only
     by accident).

8) SETUP: Missing env templates (README referenced .env.sample that didn't exist)
   New files:
     notesphere-backend/.env.sample  (PORT, MONGODB_URL, JWT_SECRET, CLIENT_URL)
     notesphere/.env.sample          (REACT_APP_API_URL)

9) SETUP: Backend had no run scripts
   File: notesphere-backend/package.json
   - Added "start": "node index.js" and "dev": "nodemon index.js".

10) DOCS: README was inaccurate
    File: README.md
    - Fixed wrong env var name: MONGO_URI -> MONGODB_URL (the app reads
      MONGODB_URL; the old name connected to nothing).
    - Rewrote the setup steps to reflect the TWO separate apps
      (backend on :5000, frontend on :3000) with correct install/env/start

PHASE 7 — ABOUT PAGE "N" VISIBILITY / GHOST-N FIX
================================================================================
Problem: the "N" of "About NoteSphere" was invisible in both modes, and an
earlier attempt produced a duplicate/ghost "N" on the left.

Root cause: the "N" carries transform: rotate(28deg). A transformed element (or
a block <div>) does NOT inherit a parent's `background-clip: text` gradient, so
the rotated N got a transparent fill with no gradient behind it (invisible).
Giving it its own clip WHILE the parent also clipped caused the ghost.

31) File: notesphere/src/components/pages/About.css  (CSS-only fix)
    - Made the <h1> a SOLID theme color (var(--text)) so "About" is never
      transparent and adapts to light/dark.
    - Gave the "N" (#full) and "oteSphere" (.n) EACH their own gradient fill
      (single clip per element) -> both visible in light and dark, no ghost N.
    - The heading markup (About.js) was left unchanged.

  committed TEMPLATE and is never read by the code. Recommendation: keep only
  placeholder values in .env.sample (avoid committing your real JWT secret).

SETUP & RUN INSTRUCTIONS (step by step)
================================================================================

--------------------------------------------------------------------------------
A) PREREQUISITES (install once)
--------------------------------------------------------------------------------
  1. Node.js (v16 or newer)         -> https://nodejs.org   (includes npm)
     Check:  node -v   and   npm -v
  2. MongoDB Community Server        -> https://www.mongodb.com/try/download/community
     (installs a Windows service named "MongoDB Server (MongoDB)")
  3. (Optional) MongoDB Compass      -> GUI to view the database
     NOTE: Compass is only a VIEWER. It needs the MongoDB server running; it
     does not start one itself.

--------------------------------------------------------------------------------
B) FIRST-TIME SETUP (do once)
--------------------------------------------------------------------------------
  This repo has TWO apps. Set up each one.

  BACKEND (notesphere-backend):
  -----------------------------
  1. Open a terminal in the "notesphere-backend" folder.
  2. Install dependencies:
        npm install
  3. Create the real environment file from the template:
        copy .env.sample .env      (Windows CMD)
        cp   .env.sample .env       (Git Bash / PowerShell)
  4. Open .env and set your real values:
        PORT=5000
        MONGODB_URL=mongodb://localhost:27017/noteSphere
        JWT_SECRET=<any long random secret string>
        CLIENT_URL=http://localhost:3000
     ( .env is gitignored and is the file the app actually reads.
       .env.sample is only a committed template and is never read. )

  FRONTEND (notesphere):
  ----------------------
  1. Open a terminal in the "notesphere" folder.
  2. Install dependencies:
        npm install
  3. (Optional) create a frontend .env from the template:
        copy .env.sample .env
     It only sets REACT_APP_API_URL=http://localhost:5000 . You can skip this
     because the code already defaults to that URL for local development.

--------------------------------------------------------------------------------
C) RUNNING THE APP (every time) — needs 3 things, in this order
--------------------------------------------------------------------------------
  STEP 1 — Start MongoDB (needs Administrator):
     Option A (command line):
        - Press Windows key, type "PowerShell", right-click -> Run as
          administrator -> Yes
        - Run:   net start MongoDB
        - Success: "The MongoDB Server (MongoDB) service was started successfully."
     Option B (GUI):
        - Press Win+R, type  services.msc , Enter
        - Find "MongoDB Server (MongoDB)" -> right-click -> Start

  STEP 2 — Start the backend (in its own terminal):
        cd notesphere-backend
        npm run dev
     Success looks like:
        iNotebook at listening at http://localhost:5000
        Connected to mongoDB successfully!
     (Use `npm start` instead of `npm run dev` for a non-auto-restarting run.)

  STEP 3 — Start the frontend (in a SECOND terminal, keep backend running):
        cd notesphere
        npm start
     It opens http://localhost:3000 automatically.
     (In VS Code, use the split-terminal button to run backend + frontend
      side by side.)

  Then: open http://localhost:3000 , sign up, log in, and manage your notes.
  In Compass, connect with:  mongodb://localhost:27017
  Your data appears under the database "noteSphere" -> collection "notes".

--------------------------------------------------------------------------------
D) TROUBLESHOOTING
--------------------------------------------------------------------------------
  * "connect ECONNREFUSED 127.0.0.1:27017" (backend OR Compass):
      MongoDB server is not running. Do STEP 1 above (start it as Administrator).
      Verify it is listening: nothing else needed once `net start MongoDB` says
      "started".

  * "Access is denied" when running `net start MongoDB`:
      The terminal is not elevated. Re-open PowerShell as Administrator.

  * MongoDB service won't start at all:
      Stop it, delete a stale lock file, start again (all as Administrator):
        net stop MongoDB
        del "C:\Program Files\MongoDB\Server\*\data\mongod.lock"
        net start MongoDB
      (Adjust the path if your data directory is elsewhere, e.g. C:\data\db.)

  * Frontend can't reach the backend after deploying elsewhere:
      Set REACT_APP_API_URL in notesphere/.env to the backend's URL, and set
      CLIENT_URL in notesphere-backend/.env to the frontend's URL (for CORS).

  * "Too many attempts. Please try again after 15 minutes." on login/signup:
      That's the rate limiter (10 attempts / 15 min per IP). Wait, or raise the
      limit in notesphere-backend/middleware/rateLimiter.js during testing.

  * Dark mode: click the moon/sun icon at the top-right of the navbar. Your
      choice is remembered across reloads (localStorage key "theme").
